# ✂️ אורי גרושקו — מערכת תורים למספרה

אפליקציית תורים (PWA) לניהול וקביעת תורים במספרה, עם **צד בעלים** ששולט בכל מה
שהלקוחות רואים, ו**צד לקוחות** שמתעדכן בזמן אמת. עיצוב מודרני — שחור · לבן · תכלת.

---

## מה יש באפליקציה

### 👑 צד בעל העסק (המנהל)
- **שעות פעילות שבועיות** — לכל יום בשבוע ניתן לקבוע אם הוא פעיל, ובאילו שעות.
  השינוי מתעדכן מיד אצל כל הלקוחות.
- **יומן זמינות יומי** — בלשונית ״יומן״ בוחרים יום ורואים את כל השעות (במרווחי
  45 דק׳ כברירת מחדל). ליד כל שעה יש מתג ״פנוי / לא פנוי״ — כיבוי המתג מסתיר את
  השעה מיד אצל הלקוחות. שעה שכבר נקבעה מסומנת ״תפוס״.
- **שינוי שעות בכל רגע** — עדכון שעות הפעילות בזמן אמת.
- **ניהול שירותים** — סוג התספורת (למשל "תספורת גבר"), מחיר ומשך זמן. כל שינוי
  מתעדכן מיד אצל הלקוחות.
- **רשימת תורים** — כל התורים שנקבעו, עם שם הלקוח, טלפון (בלחיצה מתקשרים), שירות,
  תאריך ושעה. אפשר לבטל תור.
- **התראה על תור חדש** — ברגע שלקוח קובע תור, הבעלים מקבל הודעה (פוש + התראה בתוך
  האפליקציה).

### 🙍‍♂️ צד הלקוח
- **יומן זמינות** — בחירת שירות, יום ושעה פנויה. תורים תפוסים מסומנים "תפוס".
- **קביעת תור מיידית** — ברגע שלקוח קובע תור, השעה **נעלמת אוטומטית** אצל כל שאר
  הלקוחות (חשוב מאוד — אין תורים כפולים).
- **פרטי לקוח** — בקביעת תור נדרשים שם פרטי, שם משפחה ומספר טלפון נייד תקין
  (המערכת בודקת שהמספר חוקי).
- **אישור הגעה** — בכניסה לאפליקציה מוצג באנר לאישור הגעה לתור הקרוב.
- **תזכורת פוש** — האפליקציה מבקשת הרשאה לשליחת התראות, ושולחת תזכורת שעה לפני
  התור (הזמן ניתן לשינוי אצל הבעלים).

## 🚀 העלאה לאוויר (GitHub Pages) — בחינם

1. ודאו שכל הקבצים נמצאים ב-branch הראשי (`main`) של המאגר.
2. ב-GitHub: **Settings → Pages**.
3. תחת *Build and deployment* בחרו: **Source: Deploy from a branch**,
   **Branch: `main` / `root`**, ולחצו **Save**.
4. אחרי דקה האתר יעלה לכתובת:
   `https://<שם-המשתמש>.github.io/<שם-המאגר>/`

> חשוב: התראות פוש והתקנת האפליקציה למסך הבית עובדות רק דרך **HTTPS**
> (כמו GitHub Pages) — לא בפתיחת הקובץ ישירות מהמחשב.

### התקנה כאפליקציה בטלפון
נכנסים לכתובת מהטלפון → תפריט הדפדפן → **"הוסף למסך הבית"**. האפליקציה תיפתח
במסך מלא כמו אפליקציה רגילה.

## ⚙️ הגדרות בסיסיות — קובץ `config.js`

זהו הקובץ היחיד שצריך לגעת בו:

```js
window.UG_CONFIG = {
  ownerPasscode: "******",      // סיסמת הכניסה הנסתרת לצד הניהול
  defaults: {
    shopName: "אורי גרושקו",     // שם העסק (ניתן לשנות גם מתוך "הגדרות")
    slotStep: 15,               // מרווח ברירת מחדל בין תורים (דקות)
    reminderMinutes: 60,        // תזכורת כמה זמן לפני התור
  },
  firebase: { /* ראו בהמשך */ },
};
```

**כניסת מנהל (נסתרת):** האפליקציה נפתחת ישירות בתצוגת הלקוח — הלקוחות אינם רואים
כפתור ניהול. כדי להיכנס לצד הניהול מקישים **3 פעמים ברצף על ריבוע הלוגו (האות "א")**
בראש המסך, ואז נפתח חלון להזנת הסיסמה שנקבעה ב-`ownerPasscode`.
את כל שאר ההגדרות (שם, טלפון, מרווח תורים, זמן תזכורת) אפשר לשנות ישירות ממסך
**"הגדרות"** בצד הניהול.

## 🌍 סנכרון בין כל המכשירים (אופציונלי — Firebase)

כברירת מחדל האפליקציה עובדת ב**מצב מקומי**: כל השינויים מסונכרנים בזמן אמת בין
כל הכרטיסיות והמשתמשים **באותו מכשיר/דפדפן** (מצוין להתנסות ולהדגמה).

כדי שהלקוחות יראו את השינויים בזמן אמת **מהטלפונים שלהם**, והבעלים ינהל מהטלפון
שלו — צריך חיבור ענן. האפליקציה תומכת ב-**Firebase Firestore** ללא צורך בבנייה:

1. היכנסו ל-[console.firebase.google.com](https://console.firebase.google.com)
   וצרו פרויקט חדש (חינם).
2. **Build → Firestore Database → Create database** (מצב Production או Test).
3. **Project settings → General → Your apps → Web (`</>`)** — רשמו אפליקציה
   והעתיקו את אובייקט `firebaseConfig`.
4. הדביקו את הערכים בתוך `config.js`, בשדה `firebase`:

```js
firebase: {
  apiKey: "…",
  authDomain: "…",
  projectId: "…",
  storageBucket: "…",
  messagingSenderId: "…",
  appId: "…",
},
```

5. הגדירו כללי גישה ב-Firestore (**Rules**). לשימוש בסיסי:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /shops/{doc} {
      allow read, write: if true;   // פשוט להתחלה. ראו הערת אבטחה למטה.
    }
  }
}
```

לאחר מכן, האפליקציה תזהה את ההגדרות אוטומטית ותעבור למצב ענן — כל המכשירים
מסונכרנים בזמן אמת. במסך "הגדרות" של הבעלים יופיע חיווי ✅ *מחובר לענן*.

> **הערת אבטחה:** הכלל `allow write: if true` פותח כתיבה לכל אחד — מתאים לעסק
> קטן/הדגמה. לפרודקשן אמיתי מומלץ להוסיף אימות (Firebase Auth) ולהגביל את הכתיבה
> ליומן/שירותים לבעלים בלבד.

## 🔔 על התראות פוש

- בלחיצה על "אפשר התראות" נפתח **חלון ההרשאה של הדפדפן** — יש לאשר.
- אם ההרשאה נחסמה, ניתן להפעיל שוב דרך הגדרות האתר בדפדפן/הטלפון.
- תזכורות ה"שעה לפני" מתוזמנות כל עוד האפליקציה נטענה במכשיר (או פתוחה ברקע כ-PWA).
  לתזכורות ברקע מלא גם כשהאפליקציה סגורה לגמרי נדרשת הרחבה עתידית עם
  Firebase Cloud Messaging (FCM) ושרת שליחה.

## 🗂️ מבנה הפרויקט

```
index.html              — מעטפת העמוד
config.js               — הגדרות (קוד בעלים, ברירות מחדל, Firebase)
manifest.webmanifest    — הגדרות PWA
sw.js                   — Service Worker (PWA + התראות)
assets/
  styles.css            — עיצוב (שחור/לבן/תכלת, RTL)
  img/                  — אייקונים
  js/
    util.js             — עזרי תאריך/שעה
    store.js            — שכבת הנתונים המשותפת (מקומי / Firebase) + זמן אמת
    notify.js           — הרשאות והתראות פוש + תזכורות
    app.js              — מסכים, תצוגה וחיווט
```

## 🛠️ הרצה מקומית לפיתוח

מכיוון שהאפליקציה משתמשת ב-Service Worker, יש להריץ דרך שרת (לא לפתוח קובץ ישירות):

```bash
# מתוך תיקיית הפרויקט
python3 -m http.server 8080
# ואז לפתוח:  http://localhost:8080
```
